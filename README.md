### Datasets

The project includes several PyTorch `Dataset` classes designed for various self-supervised and supervised denoising tasks, all built upon a common `BasePatchDataset`. This base class handles efficient random 3D patch extraction from volumetric data (3D, 4D, or 5D tensors) and provides common transformations like rotation and augmentation.

**Input Data Conventions:**
The datasets expect input data (`primary_data` and optional `secondary_data` in `SSUnetData`) to generally follow these N-dimensional conventions, which are then processed into `(C, D, H, W)` patches:
*   **3D Volume:** `(Depth, Height, Width)` - Interpreted as single-channel, `C=1`.
*   **4D Volume:** `(Channels, Depth, Height, Width)`
*   **5D Volume:** `(Time, Channels, Depth, Height, Width)` - A random time frame is selected for each sample, and the resulting `(C,D,H,W)` volume is used for patch extraction.

**Core Patch Extraction (`BasePatchDataset`):**
*   Randomly samples 3D patches of `(config.z_size, config.xy_size, config.xy_size)`.
*   Handles time-slicing for 5D inputs.
*   Applies configured rotations and augmentations (flips, transpose of H,W).
*   Outputs patches in `(Channels, PatchDepth, PatchHeight, PatchWidth)` format.

**Derived Dataset Classes:**

1.  **`BinomDataset`:**
    *   **Purpose:** For self-supervised methods like Noise2Void where training pairs are generated by splitting input data.
    *   **Output per sample:** `[target_patch, noise_patch, (optional) ground_truth_patch]`
    *   **Mechanism:** Takes a patch from `primary_data`. Splits it into a `target_patch` and a `noise_patch` using binomial sampling based on a dynamically sampled `p-value` (configurable methods: "signal", "db", "fixed", "list").
    *   If `secondary_data` is provided, it's passed through as the `ground_truth_patch`.

2.  **`BernoulliDataset`:**
    *   **Purpose:** Similar to `BinomDataset`, but uses Bernoulli sampling for noise.
    *   **Output per sample:** `[target_patch, noise_patch (binary), (optional) ground_truth_patch]`
    *   **Mechanism:** Inherits from `BinomDataset` but overrides noise sampling to use a Bernoulli distribution, resulting in binary (0 or 1) noise values.

3.  **`PairedDataset`:**
    *   **Purpose:** For supervised training where input and target volumes are explicitly provided.
    *   **Output per sample:** `[input_patch, target_patch]`
    *   **Mechanism:** Extracts a patch from `primary_data` (input) and a spatially corresponding patch from `secondary_data` (target). Requires `secondary_data`.

4.  **`ValidationDataset`:**
    *   **Purpose:** Provides data for model validation.
    *   **Output per sample:** `[target_for_loss, original_input_patch, (optional) ground_truth_patch]`
    *   **Mechanism:** Extracts a patch from `primary_data`. The `target_for_loss` is typically a normalized version of this patch (or the patch itself if `normalize_target=False`). The `original_input_patch` is also provided. If `secondary_data` (ground truth) is available, its corresponding patch is included.

5.  **`N2NSkipFrameDataset` (Noise2Noise variant):**
    *   **Purpose:** For Noise2Noise-like training using temporally distinct frames from the same volume.
    *   **Output per sample:** `[odd_frames_patch, even_frames_patch, (optional) ground_truth_odd_frames_patch]`
    *   **Mechanism:** Extracts a deeper source patch (depth = `config.z_size * 2`). It then splits this source patch along its depth dimension into two patches: one containing odd-indexed frames and another containing even-indexed frames. If `secondary_data` is provided, its odd frames form the ground truth.

All datasets leverage `DataConfig` for patch dimensions, augmentation settings, and `virtual_size` to control epoch length.
